## Правила работы

### 🕐 Правило 0 — Дата сессии (ОБЯЗАТЕЛЬНО)
В начале каждой сессии:
1. Взять текущую дату из системы (она всегда доступна)
2. Использовать её при записи в CHAT_SUMMARIES.md, Tasks Board.md и knowledge-base
3. Никогда не использовать устаревшую дату из прошлых записей
4. Если Анатолий называет дату явно — использовать её (она приоритетна над системной)

### 📝 Правило 0б — Фиксация всего важного
Всё существенное из разговора фиксировать ДО конца сессии:
- Новые факты о бизнесе → соответствующий файл knowledge-base/
- Решения → DECISIONS_LOG.md
- Новые задачи → Tasks Board.md
- Итоги стратегических обсуждений → CHAT_SUMMARIES.md с датой сессии
- Изменения в составе команды, статусах → 06_КОМАНДА.md
Не ждать отдельной команды «зафиксируй» — делать по ходу разговора.

1. **Начало сессии** — прочитай `CLAUDE.md` → `00_DASHBOARD.md` → `MISSION_CONTROL.md`. Остальные файлы подгружай только по необходимости.
2. **Задачи** — только в `Tasks Board.md`. Колонки: Сегодня (макс 3) / На неделе / Следующая / Заморожено / Готово
3. **Новый контент** — в `telegram/` или `youtube/` с датой в имени файла.
4. **Расхождение** — немедленно сигнализируй: `⚡ РАСХОЖДЕНИЕ | Файл: [имя] | В базе: [...] | Сейчас говоришь: [...] | Обновить?`
5. **Идеи в разговоре** — предложи записать в [[IDEAS_LOG]].
6. **Важное решение принято** — зафикси в [[DECISIONS_LOG]].
7. **Obsidian-ссылки** — оборачивай в `[[...]]` все сущности из таксономии ниже во **всех** файлах базы знаний, включая операционные (`00_DASHBOARD.md`, `MISSION_CONTROL.md`, `Tasks Board.md`, `CHAT_SUMMARIES.md`). Это обязательно для работы графа связей. При редактировании любого файла — проверяй, все ли именованные сущности обёрнуты в двойные скобки.
8. **Работа с проектом** — читай `projects/[Название]/ПРОЕКТ.md`, обновляй лог решений и чеклисты внутри проекта.
9. **Устаревшие данные** (> 2 недель) — предупреждай.
10. **После операции с базой** — всегда пиши запись в `log.md`.
11. **После работы с продуктом** — пиши запись в `memory/log.md` и файл `memory/sessions/ДАТА_агент.md`. Это обязательно — без этого следующий агент не будет знать контекст.

## Паттерн Карпатия — Workflow

```
обработай → читай raw/ → выжимай инсайты → обнови 1-3 файла KB → index.md + log.md
запрос    → читай index.md → находи нужные файлы → отвечай → предложи сохранить
проверка  → воскресенье → чеклист → доклад одним блоком
```

### Протокол команды `обработай` (7 шагов)

1. Прочитать источник (файл из `raw/`, ссылка или текст)
2. Открыть `index.md`, найти 1–3 релевантных KB файла по summary-строкам
3. Обновить KB — только добавлять, не удалять; обновить `updated:` в frontmatter
4. Пометить `raw/` файл тегом `status: processed` в frontmatter
5. Обновить `index.md` если появилась новая сущность или изменился scope файла
6. Записать в `log.md`:
   ```
   ## [ДАТА] обработка | [источник] — обновлены: [файлы] — инсайт: [одна строка]
   ```
7. Доложить результат: что прочитано, куда записано, главный инсайт
