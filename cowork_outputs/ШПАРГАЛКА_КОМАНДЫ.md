## 1. `обработай [файл или ссылка]`

Читает источник → выжимает инсайты → обновляет 1–3 KB-файла → помечает `status: processed` → пишет в `log.md`.

**Пример — статья о рынке:**
```
обработай raw/2026-05-10_коммерсантъ-нды-ритуальные.md
```
Claude прочитает статью, обновит `12_КОНКУРЕНТЫ.md` новыми данными рынка и `10_СТРАТЕГИЯ.md` если есть новые риски. Запишет в `log.md`: источник → файлы → инсайт одной строкой.

---

## 2. `обработай raw/`

Пакетная обработка всех файлов в `raw/` без `status: processed`.

**Пример — накопилось несколько источников:**
```
обработай raw/
```
После командировки в Екатеринбург скинул 3 статьи через Web Clipper → все три обрабатываются за один проход. Каждый файл получит `status: processed`, KB-файлы обновятся, три записи появятся в `log.md`.

**Когда запускать:** раз в неделю или когда в `raw/` больше 2 необработанных файлов.

---

## 3. `проверь систему`

Запускает 7-пунктный lint-чеклист здоровья vault → выдаёт доклад одним блоком.

**Пример:**
```
проверь систему
```
Claude проверит: не завис ли `inbox/`, не устарели ли KB-файлы, нет ли задачи от прошлой недели в «Сегодня», есть ли решения без результата в `DECISIONS_LOG`, не пуст ли `IDEAS_LOG`, нет ли мусора в корне vault.

**Когда запускать:** каждое воскресенье утром перед планированием недели.

---

## 4. `разбери inbox`

Читает каждый файл в `inbox/` → предлагает: куда переместить, обработать или архивировать.

**Пример — после отпуска:**
```
разбери inbox
```
Вернулся из отпуска, в `inbox/` накопились PDF-договоры, скриншоты конкурентов, фото с выставки. Claude разберёт: договоры → `_archive/legal/`, фото конкурентов → `raw/` для обработки, фото работ → `projects/Редизайн_сайта/Фото памятников/`.

**Когда запускать:** раз в неделю или когда в `inbox/` больше 5 файлов.

---

## 5. `обнови метрики`

Задаёт уточняющие вопросы по текущим цифрам → обновляет `MISSION_CONTROL.md`.

**Пример — начало недели:**
```
обнови метрики
```
Claude спросит: «Сколько заказов закрыто за апрель? Средний чек? Есть ли новые данные по [[Оператор ЧПУ]]?» Ты отвечаешь цифрами — Claude обновляет трекер недели, записывает в `MISSION_CONTROL.md` и `05_ФИНАНСЫ.md` если есть новые данные.

**Когда запускать:** каждый понедельник или после закрытия нескольких крупных заказов.

---

## 6. `план недели`

Читает `Tasks Board.md` + `MISSION_CONTROL.md` → формирует конкретный план с приоритетами.

**Пример — понедельник утром:**
```
план недели
```
Claude увидит: [[Редизайн сайта]] — дедлайн через 16 дней, подрядчик не выбран 🔴; задача по отзывам [[Карточка Яндекс Карты]] висит с 21.04 🟡; [[Оператор ЧПУ]] — ждём офлайн-откликов. Выдаст: «Сегодня: позвонить 3 клиентам за отзывами. На неделе: определиться с подрядчиком сайта. Следующая: найти внедренца [[1С Ритуальные услуги]]».

**Когда запускать:** каждый понедельник, 5 минут до начала рабочего дня.

---

## 7. `итоги недели`

Спрашивает что сделано → перемещает задачи в «Готово» → обновляет `DECISIONS_LOG.md` и `MISSION_CONTROL.md` → предлагает сохранить саммари.

**Пример — пятница вечером:**
```
итоги недели
```
Claude спросит: «Закрыли заказы? Продвинулся [[Редизайн сайта]]? Есть результат по найму [[Оператор ЧПУ]]?» Ты отвечаешь — Claude переносит выполненное в «Готово», добавляет строку «Статус» к решениям в `DECISIONS_LOG.md`, предложит сохранить саммари в `CHAT_SUMMARIES.md`.

**Когда запускать:** каждую пятницу или воскресенье.

---

## 8. `новый проект [название]`

Создаёт папку `projects/Название/ПРОЕКТ.md` по шаблону → обновляет `index.md` → пишет в `log.md`.

**Пример — новая услуга:**
```
новый проект Уход за захоронением
```
Claude создаст `projects/Уход_за_захоронением/ПРОЕКТ.md` — уже с данными из `10_СТРАТЕГИЯ.md` (там концепция описана) и `12_КОНКУРЕНТЫ.md` (незанятая ниша, иногородние клиенты). Обновит `index.md` — проект появится в секции PROJECTS со статусом 🟡 Планирование.

**Когда запускать:** при старте любой новой инициативы, которая потребует более одного шага.

---

## 9. `сохрани это`

Сохраняет ценный ответ Claude в нужное место vault — `knowledge-base/`, `projects/` или `cowork_outputs/`.

**Пример — после разбора возражения:**
```
сохрани это
```
После того как разобрали новое возражение «у конкурентов дешевле на 30%» с формулировками — Claude предложит: добавить блок в `СКРИПТ_ПРОДАЖ.md` или сохранить в `cowork_outputs/продажи/2026-04-29_возражение-цена.md`.

**Когда запускать:** после любого разговора, из которого вышло что-то конкретное и повторно используемое.
