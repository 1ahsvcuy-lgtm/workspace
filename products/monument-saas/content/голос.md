## 1. ФОКУС

**Целевой пользователь:** Олег — владелец мастерской памятников на 1–3 точки.
Сам отвечает за деньги, приёмщиков, цех и ошибки. Не теоретик — практик с опытом Excel, бумаги и 1С.

**Критический путь «работа сделана»:**

> Приёмщик заполнил параметры комплекса → нажал «Проверить комплекс» → увидел список скрытых расходов с объяснениями → получил безопасную цену и уровень риска → **ушёл к клиенту с цифрой, а не с догадкой.**

Это 8 шагов из раздела 5 Чертежа MVP. Ничего сверх.

Момент «Ага»: экран №3 — «Обнаружено 11 скрытых расходов». Именно здесь продукт доказывает ценность.

## 2. ГОЛОС — правила тона

### Правило 1. Говорим как мастер, не как маркетолог
Олег слышал достаточно красивых презентаций. Он верит числам и конкретным ситуациям, а не слоганам.

❌ «Умная система автоматизации вашего бизнеса»
✅ «Нашли 11 позиций, которые не были заложены в смету»

### Правило 2. Называем деньги и причины
Каждое добавленное действие или материал сопровождается суммой и коротким объяснением. Никаких «позиций добавлено» без контекста.

❌ «Добавлена арматура»
✅ «Добавлено 15 м арматуры — вес комплекса превышает 300 кг»

### Правило 3. Предупреждаем, не рекламируем
Тон системы — профессиональный контролёр, а не продавец. Система говорит: «здесь риск», а не «посмотрите, какая мы молодцы».

❌ «Отличная смета! Всё учтено.»
✅ «Высокий риск. Причины: не заложена гидроизоляция, доставка 70 км не покрыта сметой.»

### Правило 4. Короткие фразы — без терминологии
Олег занят. Он читает по диагонали. Каждый экран — одна мысль, одно действие.

❌ «Для завершения процесса верификации параметров необходимо нажать кнопку инициации расчёта»
✅ «Проверить комплекс»

### Правило 5. Скептик должен поверить цифре, а не слову
Если система что-то утверждает — она показывает, откуда это взялось. Иначе Олег не доверяет.

❌ «Потенциальные риски выявлены»
✅ «Потенциально неучтённые расходы: 43 800 ₽ — арматура, гидроизоляция, ручная разгрузка»

## 3. СЛОВАРИК — говорим / не говорим

### Говорим
| Термин | Пример использования |
|---|---|
| скрытые работы | «Найдено 5 скрытых работ» |
| забытые позиции | «Система нашла забытые позиции» |
| расчёт комплекса | «Расчёт комплекса готов» |
| безопасная цена | «Безопасная цена: 87 000 ₽» |
| себестоимость | «Себестоимость: 54 200 ₽» |
| риск заказа | «Уровень риска: высокий» |
| смета | «Итоговая смета для цеха» |
| маржа | «Маржа под угрозой» |
| цех | «Передать в цех» |
| приёмщик | «Добавьте приёмщика» |
| проверить комплекс | (кнопка) |
| причина | «Причина: вес превышает 300 кг» |

### Не говорим
| Запрещено | Почему |
|---|---|
| ERP | вызывает тревогу — «дорого, сложно» |
| CRM | не то, зачем пришёл Олег |
| платформа | пустое слово без смысла |
| решение | маркетинговый штамп |
| внедрение | пугает сложностью |
| интеграция | не нужна в MVP |
| миллион функций | прямая цитата страха аватара |
| автоматизация | абстракция, не конкретика |
| оптимизация | то же |
| продукт | говорим «система» или просто «АнтиМинус» |
