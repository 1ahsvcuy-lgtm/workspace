# Предварительное ТЗ — «АнтиМинус»

> Составлено по итогам брейншторм-интервью. Источники: Чертёж MVP, Карта инсайта 2.0, AI-двойник Аватара.
> Дата: 2026-06-25

---

## 1. Что строим (1 абзац)

АнтиМинус — инструмент защиты маржи для владельца мастерской памятников. Приёмщик вводит состав мемориального комплекса, система автоматически находит скрытые работы, материалы и зависимые расходы, которые человек забывает посчитать, объясняет каждое добавление и показывает владельцу реальную себестоимость, безопасную цену и уровень риска — до того, как заказ подписан. Целевая фраза клиента: «Вижу всё, что заложено, понимаю риск и не продаю в минус».

---

## 2. Фокус прототипа

- **Пользователи:** два типа — владелец (Олег) и приёмщик
- **Путь:** один счастливый — приёмщик собирает комплекс → система находит скрытые позиции → Олег видит риск
- **Форм-фактор:** PWA, адаптивный — телефон, планшет, ноутбук (без установки)

## 4. Функции (детально)

---

### Ф1 — Сборщик комплекса

**Что закрывает:** приёмщик получает полный контекст комплекса для запуска расчёта.

**Правила:**

- КОГДА приёмщик нажимает «Новый комплекс» → СИСТЕМА предлагает выбрать шаблон (одиночный, двойной, семейный...) или собрать вручную
- КОГДА выбран шаблон → СИСТЕМА предзаполняет типовой набор элементов, приёмщик редактирует под клиента
- КОГДА приёмщик добавляет элемент вручную → СИСТЕМА показывает форму: тип элемента, размеры, материал (обязательно), + доп. параметры (опционально)
- КОГДА заполнены только обязательные поля → СИСТЕМА допускает быструю оценку с пометкой «неполный расчёт»
- КОГДА заполнены все поля → СИСТЕМА запускает точный расчёт

**Данные, которые вводит приёмщик:**

*Быстрый уровень (обязательно):*

- Тип элемента (памятник, тумба, цветник, оградка, плитка, др.)
- Размеры (высота × ширина × толщина, в см)
- Материал (гранит, мрамор, габбро, бетон, металл...)

*Точный уровень (дополнительно):*

- Размеры участка
- Тип грунта
- Способ монтажа
- Демонтаж (есть / нет)
- Доступ техники (есть / нет)
- Дальность доставки (км)

**Связи:** Экран 4 (Новый комплекс) → Экран 5 (Параметры) → запуск Ф2

**Флаг: 🟢 Легко** — стандартные формы, списки, шаблоны.

---

### Ф2 — Детектор скрытых работ

**Что закрывает:** система автоматически находит то, что приёмщик забыл посчитать.

**Правила:**

- КОГДА приёмщик нажимает «Проверить комплекс» → СИСТЕМА мгновенно прогоняет параметры через набор активных правил
- КОГДА правило срабатывает → СИСТЕМА добавляет позицию в список найденных с шаблонным объяснением («добавлена арматура — причина: вес комплекса превышает 300 кг»)
- КОГДА найдены скрытые позиции → СИСТЕМА показывает Экран 6 со списком и общей суммой неучтённых расходов
- КОГДА приёмщик нажимает «Убрать» на позиции → СИСТЕМА показывает предупреждение «Вы убираете рекомендованную системой позицию» и требует подтверждения
- КОГДА Олег открывает Настройки → Правила → СИСТЕМА показывает полный список правил с галочками вкл/выкл

**Примеры встроенных правил:**

- вес комплекса > 300 кг → добавить арматуру (15 м.п.)
- обнаружена подрезка плитки (периметр > X м) → добавить полировку торцов
- доставка > 50 км → добавить транспортную надбавку
- тип грунта «глина» → добавить усиление основания
- монтаж на кладбище → проверить доступ техники, если нет → добавить ручную разгрузку

**Данные:**

- Вход: параметры комплекса из Ф1
- Выход: список найденных позиций (название, количество, единица, цена из прайса, объяснение, флаг «убрана приёмщиком»)

**Связи:** Ф1 (параметры) → движок правил → Экран 6 → Ф3 (расчёт риска)

**Флаг: 🟢 Легко** — правила как список условий, движок без AI, объяснения — шаблонный текст.

---

### Ф3 — Контроль риска заказа

**Что закрывает:** владелец видит реальную картину по деньгам до подписания.

**Правила:**

- КОГДА расчёт завершён → СИСТЕМА вычисляет себестоимость (сумма всех позиций по прайсу)
- КОГДА вычислена себестоимость → СИСТЕМА применяет коэффициенты наценки и строит безопасную цену
- КОГДА безопасная цена готова → СИСТЕМА определяет уровень риска и показывает Экран 7
- КОГДА риск «красный» → СИСТЕМА автоматически отправляет email Олегу

**Расчёт безопасной цены:**

- По умолчанию: себестоимость × базовый коэффициент (единый, зашит разработчиком)
- Расширенный режим: Олег задаёт отдельные коэффициенты по категориям (материалы / монтаж / доставка)
- Минимум ручного труда: система работает сразу без настройки коэффициентов

**Уровни риска:**

| Уровень | Цвет | Условие (пример)                                       |
| -------------- | -------- | ------------------------------------------------------------------- |
| Низкий   | 🟢       | Маржа ≥ целевой % Олега                           |
| Средний | 🟡       | Маржа ниже нормы, но в плюсе                  |
| Высокий | 🔴       | Маржа < X% или есть неучтённые позиции |

**Отображение риска:** цвет + процент маржи + список причин (например: «не учтена гидроизоляция», «доставка 70 км не покрыта сметой»)

**Данные:**

- Себестоимость (₽)
- Безопасная цена (₽)
- Предполагаемая прибыль (₽ и %)
- Уровень риска (низкий / средний / высокий)
- Причины риска (список)

**Связи:** Ф2 (список позиций) → Экран 7 (риск) → Экран 8 (смета) → выгрузка PDF/Excel

**Флаг: 🟢 Легко** — арифметика + условная логика.

## 5. Карта данных

### Сущности

**Мастерская (Workshop)**

- id, название, owner_id, настройка видимости расчётов (открытая / закрытая)

**Пользователь (User)**

- id, email, пароль (хэш), роль (owner / receiver), workshop_id, имя

**Шаблон комплекса (Template)**

- id, workshop_id, название, список типовых элементов

**Комплекс (Complex)**

- id, workshop_id, creator_id, название / имя клиента [НУЖНО УТОЧНИТЬ], дата создания, статус (черновик / рассчитан)

**Элемент комплекса (Element)**

- id, complex_id, тип, размеры (В×Ш×Т), материал, доп. параметры

**Параметры объекта (ObjectParams)**

- complex_id, размер участка, тип грунта, монтаж, демонтаж, доступ техники, дальность доставки

**Найденная позиция (DetectedItem)**

- id, complex_id, rule_id, название, количество, единица, цена, объяснение, убрана (bool), кем убрана

**Правило (Rule)**

- id, workshop_id, название, условие (шаблон), действие (что добавить), шаблон объяснения, активно (bool)

**Прайс-позиция (PriceItem)**

- id, workshop_id, название, категория (материал / монтаж / доставка / прочее), цена за единицу, единица (м², м³, м.п., шт.)

**Расчёт (Calculation)**

- id, complex_id, себестоимость, безопасная цена, прибыль (₽ и %), уровень риска, причины риска (список)

**Настройки маржи (MarginSettings)**

- workshop_id, базовый коэффициент, коэффициент материалов, коэффициент монтажа, коэффициент доставки

### Где хранится

Всё в одной базе данных (серверная сторона). Прайс и правила — в настройках мастерской. Расчёты — привязаны к мастерской и пользователю.

### Моки для прототипа

- 1 мастерская, 1 владелец, 1–2 приёмщика
- 5–7 правил (арматура, гидроизоляция, фаски, доставка, ручная разгрузка, полировка, подрезка)
- Прайс: 10–15 позиций с условными ценами
- 2–3 шаблона комплексов (одиночный, двойной, семейный)
- 3–5 готовых расчётов в истории

## 7. Роли и правила

### Роли

| Действие                                       | Владелец (Олег) | Приёмщик                        |
| ------------------------------------------------------ | --------------------------- | --------------------------------------- |
| Создать расчёт                            | ✅                          | ✅                                      |
| Видеть свои расчёты                   | ✅                          | ✅                                      |
| Видеть чужие расчёты                 | ✅ всегда             | Настраивается Олегом |
| Убрать найденную позицию         | ✅                          | ✅ (с предупреждением)  |
| Удалить расчёт                            | ✅                          | ❌                                      |
| Редактировать прайс                  | ✅                          | ❌                                      |
| Вкл/выкл правила                         | ✅                          | ❌                                      |
| Настроить наценки                      | ✅                          | ❌                                      |
| Добавить/удалить приёмщика     | ✅                          | ❌                                      |
| Настроить видимость расчётов | ✅                          | ❌                                      |

### Валидация

- Нельзя запустить «Проверить комплекс» если не выбран ни один элемент
- Нельзя сохранить элемент без обязательных полей (тип + размеры + материал)
- При удалении найденной позиции — обязательное подтверждение с предупреждением
- Прайс пустой → система предупреждает Олега, расчёт выдаёт нули с пометкой «заполните прайс в настройках»

## 8. Состояния экранов

### Экран «Список расчётов»

- **Пустое:** «Расчётов пока нет. Создайте первый комплекс →»
- **Загрузка:** не нужна (мгновенно)
- **Ошибка:** «Не удалось загрузить расчёты. Проверьте соединение.»

### Экран «Параметры объекта»

- **Быстрый режим:** только обязательные поля, кнопка «Добавить подробности»
- **Ошибка:** подсветка пустых обязательных полей, расчёт не запускается

### Экран «Скрытые позиции»

- **Ноль найденных:** «Скрытых расходов не обнаружено. Расчёт чистый.»
- **Есть позиции:** список с суммой, кнопка «Подтвердить и перейти к риску»
- **Прайс не заполнен:** суммы — нули, предупреждение «Заполните прайс для расчёта стоимости»

### Первый запуск (онбординг-мастер)

- **Шаг 1:** Добавьте приёмщиков (email + имя, можно пропустить)
- **Шаг 2:** Заполните прайс — хотя бы 5 позиций для старта
- **Шаг 3:** Создайте первый комплекс — кнопка «Поехали»

## 10. Метрика «работа сделана»

**Главная метрика (из Чертежа):**

> Количество найденных скрытых позиций до подписания договора

**Критерий покупки (из Чертежа):**

> «Без неё теперь не считаю комплексы» или «Она несколько раз поймала то, что пропустил бы человек»

**Вторичные метрики:**

- Сколько заказов получили предупреждение о красном риске
- Сколько позиций добавлено автоматически (не убрано приёмщиком)
- Сколько раз открыли блок «Почему добавлено»
- Сколько комплексов пересчитали после предупреждения

## ❓ Открытые вопросы

> Статус обновлён по данным NotebookLM (27 июня 2026).

1. **Название комплекса.** `[ЗАКРЫТО]`  
   Поле «Имя клиента» — **обязательно**. Ошибки в ФИО при передаче в цех ведут к браку стелы. Идентификация: имя клиента + дата создания. Поле при создании — рекомендованное, но не блокирующее.

2. **Пороговые значения правил.** `[ЧАСТИЧНО ЗАКРЫТО]`  
   - Арматура: вес комплекса > **300 кг** → добавить арматуру ✅  
   - Транспортная надбавка: дальность > **70 км** → добавить надбавку ✅  
   - Подрезка плитки → полировка торцов: **нет данных** `[НУЖНО УТОЧНИТЬ]`

3. **Базовый коэффициент маржи.** `[ЗАКРЫТО]`  
   Норма рынка РФ/СНГ: **20–30% чистой маржи**. По умолчанию зашить **25%** как середину диапазона.

4. **Уровни риска — пороги.** `[ПРЕДПОЛОЖЕНИЕ — подтвердить с Олегом]`  
   Данных в источниках нет. Рабочая гипотеза для прототипа:  
   - 🟢 Низкий: маржа ≥ 25%  
   - 🟡 Средний: маржа 15–24%  
   - 🔴 Высокий: маржа < 15%

5. **Шаблоны комплексов.** `[НЕТ ДАННЫХ — заглушки для прототипа]`  
   Покомпонентный состав нигде не раскрыт. Условный состав для прототипа:  
   - Одиночный: стела + цветник + плитка  
   - Двойной: 2 стелы + цветник + ограда  
   - Семейный: несколько стел + плитка + благоустройство

6. **Единицы измерения в прайсе.** `[ЗАКРЫТО]`  
   - **м²** — квадратура камня, облицовка, укладка плитки  
   - **м³** — кубатура каменных элементов (объём)  
   - **м.п.** — резка и полировка нестандартных форм, установка ограды  
   - **шт.** — буквы при гравировке, 3D-модели (вазы, лампады, столики), мешки цемента, пироны

7. **Валидация на живых Олегах.** `[ОСТАЁТСЯ ОТКРЫТЫМ]`  
   Пороги правил (вопросы 2, 4, 5) — из исследований и предположений. Требуют проверки на реальных владельцах мастерских до запуска рабочей версии.
