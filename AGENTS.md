# AGENTS.md — единый протокол работы в WORKSPACE

> Канонические правила для любого AI-агента. Проектные `AGENTS.md` дополняют этот файл, но не отменяют его.
> Обновлено: 2026-06-27 23:29 +05:00 | Агент: Codex

## Быстрый старт — читать минимум

1. `AGENTS.md` — эти правила.
2. `memory/STATE.md` — текущее состояние и следующие действия.
3. Только нужный раздел `index.md`.
4. Для продукта/проекта: его `README.md` или `ПРОЕКТ.md`, `ПРОГРЕСС.md` и локальный `AGENTS.md`.
5. `memory/decisions.md` и последнюю релевантную запись `memory/sessions/` — только если задача требует истории решения.

Не читать весь vault подряд. Искать через `rg`, открывать только релевантные фрагменты. Это основной механизм экономии токенов.

## Источники правды

- Навигация: `index.md`.
- Текущее состояние и незавершённая работа: `memory/STATE.md`.
- Задачи бизнеса: `Tasks Board.md`; задачи продукта: также его `ПРОГРЕСС.md`.
- Решения: `DECISIONS_LOG.md` для бизнеса, `memory/decisions.md` для продуктов и системы.
- Факты бизнеса: соответствующий файл `knowledge-base/`.
- История действий: `memory/log.md`; подробности значимой сессии: `memory/sessions/`.
- Проектная специфика: локальный `AGENTS.md` и файлы проекта.

При расхождении сначала отметить его, затем обновить устаревший производный файл. Не размножать один и тот же подробный текст: хранить его в одном источнике правды, в остальных — краткий статус и `[[ссылка]]`.

## Обязательная фиксация по ходу работы

Агент не ждёт команды «зафиксируй». До завершения ответа он обновляет все затронутые источники правды:

- выполненная работа → файл проекта/продукта + его `ПРОГРЕСС.md`;
- новая или изменённая задача → `Tasks Board.md` и, если локальная, `ПРОГРЕСС.md` проекта;
- принятое решение → соответствующий лог решений;
- новый устойчивый факт → релевантный файл `knowledge-base/`;
- изменение текущего состояния или следующего шага → `memory/STATE.md`;
- любая правка базы/продукта → одна компактная запись в `memory/log.md`;
- значимая сессия, передача работы или сложное обсуждение → `memory/sessions/YYYY-MM-DD_HHMM_<agent>.md`;
- изменение состава/назначения файлов → `index.md`.

Не создавать сессионный файл для простого чтения без изменений. Не записывать предположения как факты; маркировать `гипотеза`, `нужно подтвердить` или `решение`.

## Подпись и время

Каждая новая запись о работе, задаче, решении или статусе должна содержать:

`YYYY-MM-DD HH:MM +05:00 | Агент: <имя> | <событие>`

Время брать из системы в часовом поясе workspace (`Asia/Yekaterinburg`, UTC+05:00). Имя агента указывать явно: `Codex`, `Claude`, `Hermes` и т. п. При изменении существующей задачи добавлять короткую строку `Обновлено: ...`, а не стирать историю авторства.

## Память

- Кратковременная: текущий диалог + `memory/STATE.md` (не более ~120 строк).
- Долговременная: `memory/decisions.md`, `memory/log.md`, `memory/sessions/`, проектные и KB-файлы.
- После существенной работы сжать результат в `STATE.md`: статус, что сделано, блокеры, следующие 1–3 действия, ссылки.
- Когда `STATE.md` разрастается, переносить детали в сессию/проект, оставляя только актуальный срез.

## Завершение работы

Перед ответом пользователю проверить:

1. Изменены ли реальные исходные файлы, а не только лог.
2. Синхронизированы ли статус и задачи.
3. Есть ли подпись агента и системное время.
4. Обновлён ли `STATE.md`.
5. Добавлена ли одна запись в `memory/log.md`.

Изменения автоматически синхронизируются задачей `WORKSPACE Git Sync` через GitHub. Агент не запускает параллельный `git push`, если работает планировщик; перед завершением проверяет отсутствие конфликтов в `git status`. Устройство синхронизации описано в `scripts/sync/README.md`.

Соблюдать Obsidian-ссылки и таксономию из `CLAUDE.md`. Не записывать секреты, пароли и токены в память или логи.
