# Автоматическая синхронизация WORKSPACE

> Обновлено: 2026-06-27 23:38 +05:00 | Агент: Codex

## Архитектура

GitHub `origin/main` — центральный узел и внешняя копия. Каждый компьютер и будущий VPS имеют собственный Git-клон и каждые 5 минут выполняют один алгоритм: локальный коммит → fetch → безопасное объединение → push.

Скрипты никогда не делают `reset --hard` и не перезаписывают конфликт. Одновременные изменения разных файлов объединяются автоматически; конфликт одного файла сохраняется и блокирует дальнейшую синхронизацию до ручного решения. Локальный журнал находится в `.sync/sync.log` и не коммитится.

## Windows

Задача планировщика устанавливается командой:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\sync\install-windows-task.ps1
```

Проверка:

```powershell
Get-ScheduledTask -TaskName "WORKSPACE Git Sync"
Get-Content .\.sync\sync.log -Tail 20
```

## Будущий VPS (Linux)

Потребуются адрес VPS, пользователь и SSH-доступ к приватному GitHub-репозиторию. После клонирования в `/srv/workspace`:

1. Создать системного пользователя `workspace` и выдать ему права на `/srv/workspace`.
2. Настроить deploy key с правом записи в GitHub.
3. Скопировать `.service` и `.timer` в `/etc/systemd/system/`.
4. Выполнить `systemctl daemon-reload` и `systemctl enable --now workspace-sync.timer`.

## Восстановление

На новом компьютере или сервере достаточно клонировать `git@github.com:1ahsvcuy-lgtm/workspace.git`. Вся история версий сохраняется в Git. Для независимости от одного провайдера рекомендуется позже добавить второй удалённый backup-репозиторий или ежедневный зашифрованный snapshot.
